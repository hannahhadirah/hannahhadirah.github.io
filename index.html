Multi-screen food tracking app with navigation & local storage

Problem: Food waste happens when people do not track their food expiry properly and end up throwing away food that is still edible.

Solution: Basic food tracking system with simple navigation and local storage.

Project 2 — FreshKeeper Enhanced

Full-stack app with Room, Firebase, Retrofit API & GPS sensor

Problem: Food waste is still happening a lot because people forget expiry dates and do not manage food properly at home.

Solution: This app helps users track food, check expiry status, get recipes using API, and donate extra food using Firebase. It also uses GPS to help users find nearby food donation centres.
▶ Watch VSR on YouTube
💬 Lessons Learned
In this project, I learned how to combine Room Database, Firebase, Retrofit API and GPS sensor into one full application. It was challenging but I gained a lot of experience in building real world mobile apps.

💭 Reflection & Learning Journey

From Lab 1 until Project 2, I can clearly see my improvement in mobile development. At first, I only knew basic UI design but now I can build a full application with database, API and sensor features.

The most difficult part was integrating Firebase and API because it required proper understanding of data flow. However, through practice, I managed to solve it.

Overall, this course helped me understand how real mobile applications are built and how technology can be used to solve real world problems like food waste.

This journey also taught me that SDG 12 — Responsible Consumption and Production — is something technology can genuinely support. FreshKeeper is proof that a simple mobile app can encourage better habits, reduce waste, and even connect communities through food sharing.
